# AI система за управление на инциденти

## Училищен проект - II срок

**Предмет:** Програмиране на ИИ  
**Клас:** 11 клас  
**Тема:** ООП на Python  
**Задание:** Задание 4 - AI система за управление на инциденти  
**Етап:** Първи етап - Анализ и дизайн

## Идея

Проектът представлява Python приложение за управление на IT инциденти. Системата ще приема информация за инциденти, ще ги записва в база данни и ще използва AI логика за предсказване на приоритет:

- `low`
- `medium`
- `high`

Целта е системата да помага при по-бързо подреждане и обработване на инциденти според тяхната важност.

## Какво съдържа първият етап

Първият етап е **анализ и дизайн**. В него са подготвени:

- идея на проекта;
- основни класове;
- структура на проекта;
- дизайн на база данни;
- AI подход за класификация;
- примерни данни за инциденти.

## Файлове за предаване

- `docs/Задание 4 - Седмица 1 - Анализ и дизайн.md` - основният документ за първи етап.
- `data/sample_incidents.csv` - примерни данни с 30 инцидента.
- `docs/Проектен пакет_ ООП на Python — 11 клас, „Програмиране на ИИ“.md` - оригиналното задание.

## Планирани основни класове

| Клас | Роля |
| --- | --- |
| `Incident` | Модел на инцидент. |
| `Technician` | Модел на техник. |
| `IncidentClassifier` | AI логика за приоритет. |
| `IncidentManager` | Управление на инциденти. |
| `EventManager` | Събития и Observer pattern. |
| `DatabaseService` | Работа с SQLite база данни. |
| `FileService` | Работа с CSV/JSON файлове. |
| `AppLogger` | Запис на логове. |

## База данни

Планирана е SQLite база данни с минимум 3 таблици:

- `incidents`
- `technicians`
- `history`

## AI подход

Първата версия ще използва rule-based AI класификатор. Той ще определя приоритет според:

- брой засегнати потребители;
- категория на инцидента;
- ключови думи в описанието;
- риск за сигурност, база данни, мрежа или достъп.

Този подход е подходящ за учебен проект, защото е лесен за обяснение и демонстрация.
