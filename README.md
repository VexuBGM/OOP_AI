# AI система за управление на инциденти

## Училищен проект - II срок

**Предмет:** Програмиране на ИИ  
**Клас:** 11 клас  
**Тема:** ООП на Python  
**Задание:** Задание 4 - AI система за управление на инциденти  
**Етап:** Финален етап - Седмица 4

## Идея

Проектът е Python приложение за управление на IT инциденти. Системата създава инциденти, предсказва техния приоритет чрез AI логика, записва ги в база данни и изпраща събития към други компоненти.

Приоритетите са:

- `low`
- `medium`
- `high`

Целта е да се демонстрират ООП, комуникация между обекти, Observer pattern, файлове, база данни, LINQ-подобен анализ и прост AI компонент.

## Какво съдържа проектът

- ООП структура с повече от 6 класа.
- `IncidentClassifier` за rule-based AI класификация.
- `EventManager` с Observer pattern.
- Callback функции и custom event args.
- SQLite база данни с таблици `incidents`, `technicians` и `history`.
- CSV/JSON четене и запис.
- Анализ чрез `filter`, `map`, `reduce`, `sorted`, `Counter` и `groupby`.
- Custom exceptions.
- Автоматична финална демонстрация.
- Конзолно меню за live промяна при защита.
- Unit тестове.

## Какво показва финалната демонстрация

1. Създава се критичен инцидент.
2. AI логиката определя приоритет `high`.
3. Изпращат се събития.
4. Няколко слушателя реагират на критично събитие.
5. Данните се записват в SQLite.
6. Зареждат се 30 примерни инцидента от CSV.
7. Показва се статистика по приоритет и категория.
8. Показва се средно време за решаване.
9. Показват се топ 5 най-сериозни инцидента.
10. Извежда се кратко обяснение на AI модела.

## Основни класове

| Клас | Роля |
| --- | --- |
| `Incident` | Модел на инцидент. |
| `Technician` | Модел на техник. |
| `IncidentClassifier` | AI логика за предсказване на приоритет. |
| `IncidentManager` | Контролер за създаване и управление на инциденти. |
| `EventManager` | Event dispatcher и Observer pattern. |
| `IncidentEventArgs` | Данни за събитията. |

| `FileService` | Работа с CSV и JSON файлове. |
| `IncidentStats` | Анализ на данни. |
| `AppLogger` | Логване на събития. |
| `NotificationService` | Симулира уведомяване на екип. |

## AI подход

`IncidentClassifier` използва rule-based логика. Той изчислява точки според:

- брой засегнати потребители;
- рискова категория: `security`, `database`, `network`;
- ключови думи като `outage`, `breach`, `data loss`, `down`, `malware`;
- средно рискови думи като `slow`, `error`, `failed`, `timeout`, `locked`.

Правила:

| Точки | Приоритет |
| --- | --- |
| 0-2 | `low` |
| 3-5 | `medium` |
| 6+ | `high` |
